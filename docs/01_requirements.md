# 要件定義書: LINE 練習参加管理システム

## 1. プロジェクト概要

### 1.1 目的

チーム・サークル等の練習案内と参加可否管理をLINE上で完結させるWebアプリケーションを開発する。管理者がLINEグループに練習案内を送信し、メンバーがLIFF（LINE Front-end Framework）アプリ上で参加・不参加を登録する。データはGoogle Sheetsで管理する。

### 1.2 背景

- 現在、練習の出欠確認はLINEグループ内のテキストベースのやりとりで行っており、集計が煩雑
- メンバー数が増えると誰が回答済みか把握しづらい
- Google Sheetsに自動集計されることで、管理者の手作業を削減したい

### 1.3 用語定義

| 用語 | 説明 |
|------|------|
| LIFF | LINE Front-end Framework。LINEアプリ内のブラウザで動作するWebアプリの仕組み |
| 管理者 | 練習の作成・案内送信・出欠確認を行うユーザー。環境変数で指定されたLINE UserIDを持つ |
| メンバー | LIFFアプリを開いて参加・不参加を登録するユーザー |
| 練習（Practice） | 日時・場所が指定された練習イベント |
| 出欠（Attendance） | 各メンバーの練習に対する参加・不参加の回答 |

## 2. ステークホルダー

| 役割 | 説明 |
|------|------|
| 管理者 | 1〜2名。練習の作成・LINE送信・出欠確認を行う |
| メンバー | 10〜50名程度。LINEグループに所属し、参加可否を登録する |

## 3. 機能要件

### 3.1 メンバー向け機能

| ID | 機能名 | 説明 |
|----|--------|------|
| F-M01 | 練習一覧表示 | 今後の練習一覧を日付昇順で表示する |
| F-M02 | 参加登録 | 各練習に対して「参加」「不参加」を登録する |
| F-M03 | 回答変更 | 登録済みの回答を変更できる（upsert） |
| F-M04 | 自分の回答状態表示 | 各練習に対する自分の現在の回答状態を表示する |

### 3.2 管理者向け機能

| ID | 機能名 | 説明 |
|----|--------|------|
| F-A01 | 練習作成 | タイトル、日付、時間、場所、備考を入力して練習を作成する |
| F-A02 | LINE案内送信 | 作成済みの練習をFlex MessageとしてLINEグループに送信する |
| F-A03 | 出欠集計表示 | 練習ごとの参加・不参加・未回答の人数と詳細を表示する |
| F-A04 | メンバー機能 | 管理者もメンバーとして参加登録ができる（F-M01〜F-M04を含む） |

### 3.3 システム機能

| ID | 機能名 | 説明 |
|----|--------|------|
| F-S01 | LIFF初期化 | LINEアプリ内でLIFF SDKを初期化し、ユーザープロフィールを取得する |
| F-S02 | 管理者判定 | LINE UserIDに基づいて管理者かどうかを判定する |
| F-S03 | Sheetsヘッダー自動初期化 | 初回起動時にGoogle Sheetsのヘッダー行を自動作成する |

## 4. 非機能要件

### 4.1 性能

- 同時利用者数: 最大50名程度
- API応答時間: Google Sheets APIの制約に依存（通常1〜3秒）
- Flex Messageの送信は管理者の手動トリガーのみ

### 4.2 可用性

- Render無料枠はスリープあり（15分間アクセスがないとスリープ、起動に10〜30秒）
- Google Sheets API: 読み取り60回/分、書き込み60回/分（プロジェクト単位）
- 月1〜4回程度の利用頻度を想定

### 4.3 セキュリティ

- 管理者APIは認証が必要（LINE UserIDベース、本番ではLIFF IDトークン検証を推奨）
- Google Sheets認証はサービスアカウント（JWT）
- LINE案内送信は `liff.shareTargetPicker()` を使用（Messaging API / Bot チャネル不要）
- CORSはフロントエンドのデプロイURLのみ許可
- `.env`ファイルはGit管理外（.gitignore）

### 4.4 コスト制約

| 項目 | 費用 |
|------|------|
| Google Sheets API | 無料 |
| LINE LIFF | 無料 |
| LINE Messaging API | 不要（shareTargetPickerに変更） |
| Vercel（フロントエンド） | 無料枠 |
| Render（バックエンド） | 無料枠（スリープあり）。常時起動は$7/月 |
| **月額合計** | **¥0〜¥1,500** |

## 5. 制約条件

- LINEアプリ内ブラウザでの動作が必須（LIFF）
- データストアはGoogle Sheetsに限定（RDBは使用しない）
- LINE外部ブラウザでのLIFF動作は保証外
- メンバーの認識は`liff.getProfile()`で取得するLINE UserIDに依存

## 6. 前提条件

- Google Cloudプロジェクトが作成済みで、Google Sheets APIが有効化されていること
- LINE Developer ConsoleでLINE Loginチャネルが作成され、LIFFアプリが登録されていること
- LIFFアプリの **shareTargetPicker** 機能が有効化されていること
